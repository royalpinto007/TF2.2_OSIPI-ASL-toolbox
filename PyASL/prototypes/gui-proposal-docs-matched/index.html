Preclinical PCASL Pipeline

Runner: pyasl.pipelines.preclinical_pcasl_pipeline.run_preclinical_pcasl_pipeline

Purpose. Full preclinical pCASL chain with proposed 2026 extensions for GUI-driven execution, batch-wise processing, co-registration, and normalization.

Proposal Prototype

This page mock shows how the new GUI and batch-mode features could be documented while staying visually consistent with the current PyASL documentation structure.

YAML Example

type: pcasl

steps:
  - module: BrukerLoader
    params: { expno: 18, procno: 1 }
  - module: SteadyStateTrim
    params: { trim: 2 }
  - module: ControlLabelSplit
    params: { control_first: true }
  - module: MotionCheck
  - module: DiffImage
  - module: ComputeM0
    params: { TR: from_meta, T1t: 1900 }
  - module: PreclinicalCoregister
    params: { reference: "M0", method: "rigid_affine" }
  - module: SlicePLDAdjust
    params: { SGap: 31, T1blood: 2800 }
  - module: PreclinicalNormalize
    params: { template: "mouse_template.nii.gz" }
  - module: SaveOutputs

Prototype GUI mapping

The proposed GUI is designed as a front-end layer over the existing YAML-driven runner. A documentation-aligned interface would expose the same concepts users already see in PyASL:

  • pipeline type selection
  • step ordering and parameters
  • new preclinical co-registration and normalization steps
  • batch dataset queues and run summaries
